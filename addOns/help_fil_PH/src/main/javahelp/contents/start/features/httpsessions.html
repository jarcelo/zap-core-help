Ang HTTP ng mga sesyon

This tool keeps track of the existing HTTP Sessions on a particular Site and allows the ZAP user to force all requests to be on a particular session. Talagang, pinahihintulutan ito ang gumagamit upang madaling ilipat ang pagitan ng aggamit ng mga sesyon sa isang sayt para makabuo ng isang bagong sesyon ng walang "nasisira " na umiiral na.

The tool is only intended for manual testing and should not be used in automation - there are much better alternatives. If you are new to ZAP automation then the best place to start is the ZAP Authentication Decision Tree (external link).

Ito ay batay sa mga konsepto ng sesyon ng mga token, kung saan ay ang HTTP ng mensahe ng mga parametro (para sa ngayon ang mga cookie lamang) kung saan pinahintulutan ang isang server ng HTTP na kumonek sa isang kahilingan na mensahe sa pamamagitan ng alin man sa nakaraang mga kahilingan o datos na naitago. In the case of ZAP, conceptually, session tokens have been classified into 2 categories: default session tokens and site session tokens. Ang default na sesyon ng mga token at ang isa sa mga gumamit na maaaring maitakda sa mga opsyon ng iskrin at ay mga token na, sa pamamagitan ng default, ay awtomatikong itinuturing na sesyon ng mga token para sa kahit anong sayt (eg. phpsessid, jsessionid, at iba pa). Ang sayt ng sesyon ng mga token at isang pagtakda sa mga token para sa isang partikular na sayt at ay karaniwang naisaayos gamit ang popup na mga menu na magagamit sa mga Params ng tab.

Ang kasangkapan na ito ay awtomatikong natutuklasan, gamit ang natukoy na sesyon ng mga token o ang awtomatikong natuklasan ng default ng sesyon ng mga token, kahit anong HTTP ng sesyon ng kahit aling umiiral sa mga komunikasyon. Ang mga natuklasang sesyon ay naipakita sa mga HTTP ng mga sesyon ng tab.

Ang gumagamit ay maaaring, gumamit ng buton na pwede sa mga HTTP ng mga sesyon ng tab, bumuo ng isang bagong sesyon ng walang nasisirang mga umiirala na, o maaaring pwersahin ang isa sa mga sesyon bilang 'aktibo'. Habang ang isang sesyon ay 'aktibo', ang lahat ng mga papalabas na mga kahilingan na naipadala sa mga kaukulang mga sayt na nabago, ang sesyon ng mga token na naisaayos na upang tumugma sesyon na aktibo. Sa paraang ito, ang gumagamit ay madaling mapipilit ang ibang mga mensahe na maging 'bahagi ng isang partikular na sesyon at pagkatapos palitan at ipadala ang mga mensahe sa iba pang sesyon.

Ang HTTP ng mga sesyon ng kasangkapan ay nakumpigura gamit ang HTTP ng mga sesyon ng mga opsyon ng iskrin.
