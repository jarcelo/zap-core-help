Sesi HTTP

This tool keeps track of the existing HTTP Sessions on a particular Site and allows the ZAP user to force all requests to be on a particular session. Pada dasarnya, ini memungkinkan pengguna untuk dengan mudah beralih di antara sesi pengguna di Situs dan membuat Sesi baru tanpa "menghancurkan" yang sudah ada.

The tool is only intended for manual testing and should not be used in automation - there are much better alternatives. If you are new to ZAP automation then the best place to start is the ZAP Authentication Decision Tree (external link).

Hal ini berdasarkan pada konsep Token Sesi, yang merupakan parameter pesan HTTP (untuk saat ini hanya Cookies) serta memungkinkan server HTTP menghubungkan pesan permintaan dengan setiap permintaan sebelumnya atau data yang tersimpan. In the case of ZAP, conceptually, session tokens have been classified into 2 categories: default session tokens and site session tokens. Token sesi default adalah salah satu yang dapat disetel pengguna di Pilihan layar dan merupakan tolak ukur secara default, yang secara otomatis dianggap token sesi untuk situs manapun (mis. phpsessid, jsessionid, dll). Token sesi situs adalah sekumpulan token untuk situs khusus dan biasanya disiapkan dengan menggunakan menu popup yang tersedia di Tab Params.

Alat ini secara otomatis mendeteksi, menggunakan token sesi yang ditentukan atau token sesi default yang terdeteksi secara otomatis, setiap sesi HTTP yang ada dalam komunikasi. Sesi yang terdeteksi ditampilkan pada HTTP Sessions Tab.

Pengguna bisa menggunakan tombol yang tersedia di HTTP Tab Sesi, buat sesi baru tanpa merusak yang telah tersedia, atau bisa dengan paksa salah satu sesi sebagai 'aktif'. Ketika sesi 'aktif', semua permintaan outbound dikirim ke Situs yang sesuai yang dimodifikasi, sesi token yang dibentuk untuk menyesuaikan sesi aktif. Dengan cara ini, pengguna dapat dengan mudah memaksa beberapa pesanan untuk menjadi 'bagian dari' sesi tertentu dan kemudian beralih dan mengirim pesan pada sesi lain.

Alat HTTP sesi dikonfigurasi menggunakan HTTP sesi pilihan layar.
