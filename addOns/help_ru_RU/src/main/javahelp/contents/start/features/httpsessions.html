HTTP-сессии

This tool keeps track of the existing HTTP Sessions on a particular Site and allows the ZAP user to force all requests to be on a particular session. По сути, это позволяет пользователю легко переключаться между пользовательскими сеансами на сайте и создавать новый сеанс без «разрушения» существующих.

The tool is only intended for manual testing and should not be used in automation - there are much better alternatives. If you are new to ZAP automation then the best place to start is the ZAP Authentication Decision Tree (external link).

Он основан на концепции токенов сеанса, которые представляют собой сообщение HTTP. параметры (пока только файлы cookie), которые позволяют HTTP-серверу соединить сообщение запроса с любыми предыдущими запросами или сохраненными данными. In the case of ZAP, conceptually, session tokens have been classified into 2 categories: default session tokens and site session tokens. Токены сеанса по умолчанию - это те, которые пользователь может установить. в Параметры screen и являются токенами, которые по умолчанию автоматически считаются токены сеанса для любого сайта (например, phpsessid, jsessionid, etc). Сайт токены сеанса - это набор токенов для определенного сайта, обычно настраивается с помощью всплывающих меню, доступных на вкладке Параметры .

Этот инструмент автоматически обнаруживает, используя определенные токены сеанса или автоматически обнаруженные токены сеанса по умолчанию, любой сеанс HTTP который существует в сообщении. Обнаруженные сеансы показаны в вкладка HTTP-сеансы .

Пользователь может, используя кнопку, доступную на вкладке HTTP-сеансы , создавать новый сеанс, не разрушая существующий, или может заставить один из сеансы как «активные». Когда сеанс «активен», все исходящие запросы, отправленные на соответствующий Сайт, изменяются, сеанс токены настраиваются в соответствии с активным сеансом. Таким образом, пользователь может легко заставить некоторые сообщения быть "частью" определенного сеанса а затем переключить и отправить сообщения в другом сеансе.

Инструмент HTTP-сеансов настраивается с использованием HTTP-сеансов. Экран параметров .
