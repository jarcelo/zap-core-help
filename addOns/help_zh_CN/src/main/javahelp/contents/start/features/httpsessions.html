HTTP会话

This tool keeps track of the existing HTTP Sessions on a particular Site and allows the ZAP user to force all requests to be on a particular session. 基本上，它允许用户在网站上的用户会话之间轻松切换，并创建一个新的会话而不会“销毁”现有会话。

The tool is only intended for manual testing and should not be used in automation - there are much better alternatives. If you are new to ZAP automation then the best place to start is the ZAP Authentication Decision Tree (external link).

它基于会话令牌的概念，它是HTTP消息参数（目前仅为cookie），允许HTTP服务器将请求消息与先前存储的任何请求或数据连接起来。 In the case of ZAP, conceptually, session tokens have been classified into 2 categories: default session tokens and site session tokens. 默认会话令牌是用户可以在选项屏幕中设置的令牌，默认情况下会自动将会话令牌视为任何网站的会话令牌（例如， phpsessid，jsessionid等）。 站点会话令牌是特定站点的一组令牌，通常使用参数选项卡中提供的弹出式菜单进行设置.

该工具使用定义的会话令牌或自动检测到的默认会话令牌自动检测通信中存在的任何HTTP会话。 检测到的会话显示在HTTP会话选项卡中.

用户可以使用HTTP会话选项卡上提供的按钮创建新会话而不破坏现有会话，也可以强制其中一个会话处于“活动”状态。 当会话处于“活动”状态时，所有发送到相应站点的出站请求都会被修改，会话令牌将被设置为与活动会话相匹配。 通过这种方式，用户可以轻松地强制某些消息成为特定会话的‘一部分’，然后在另一个会话中切换并发送消息。

HTTP会话工具使用HTTP会话选项屏幕进行配置.
